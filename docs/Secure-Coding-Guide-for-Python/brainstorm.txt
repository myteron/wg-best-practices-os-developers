An example of this type of deadlock is shown in the following article [Brownlee, 2021], which describes it as "Deadlock 1: Submit and Wait for a Task Within a Task". Submitting a task will add it to an internalThreadPoolExecutorqueue. The task will be removed from the queue when one of the worker threads becomes available, i.e., after finishing its current task.
